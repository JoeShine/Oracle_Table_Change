## 1. 系统架构

```

                     │

│      - 连接管理逻辑                    │
└────────────────────┬────────────────────┘
                     │

                     │
```

## 2. 详细设计

### 2.1 数据库连接模块 (db_connection.py)

#### 2.1.1 设计要点

- 使用Python的oracledb库连接Oracle数据库
- 支持连接池管理
- 提供连接测试、查询、执行功能
- 事务管理和异常处理

#### 2.1.2 主要接口

| 接口名 | 说明 | 参数 | 返回值 |
|-------|------|------|-------|
| connect() | 建立数据库连接 | host, port, service, username, password | bool, error |
| disconnect() | 断开连接 | 无 | 无 |
| is_connected() | 检查连接状态 | 无 | bool |
| execute_sql() | 执行SQL语句 | sql, params, commit | bool, result, error |
| get_columns() | 获取表的列信息 | table_name, schema | list |
| table_exists() | 检查表是否存在 | table_name, schema | bool |

### 2.2 Excel处理模块 (excel_handler.py)

#### 2.2.1 设计要点

- 使用openpyxl库处理Excel文件
- 支持.xlsx和.xls格式
- 文件大小和行数验证
- 多列数据读取
- 数据预览功能
- 日志和失败记录导出

#### 2.2.2 主要接口

| 接口名 | 说明 | 参数 | 返回值 |
|-------|------|------|-------|
| validate_excel_structure() | 验证Excel结构 | file_path | bool, msg, data_rows |
| validate_multi_column_structure() | 多列验证 | file_path, update_columns | bool, msg, data_rows |
| get_preview_data() | 获取预览数据 | file_path | bool, msg, preview_data |
| get_multi_column_preview() | 多列预览 | file_path, update_columns | bool, msg, preview_data |
| export_logs() | 导出日志 | logs, file_path | bool, msg |
| export_failed_records() | 导出失败记录 | records, file_path | bool, msg |
| check_duplicate_keys() | 检查重复键 | data_rows | bool, duplicates |

#### 2.2.3 常量定义

```python
MAX_FILE_SIZE = 10 * 1024 * 1024  # 10MB
MAX_ROWS = 100000                 # 10万行
MAX_PREVIEW_ROWS = 50             # 预览前50行
```

### 2.3 数据更新模块 (data_updater.py)

#### 2.3.1 设计要点

- 多列数据更新支持
- 自动备份机制
- 回滚功能
- 进度回调
- 临时表管理

#### 2.3.2 主要接口

| 接口名 | 说明 | 参数 | 返回值 |
|-------|------|------|-------|
| backup_table() | 备份表 | schema, table_name | bool, msg |
| create_temp_table_multi_column() | 创建临时表 | schema, table_name, key_column, update_columns | bool, msg |
| import_excel_data_multi_column() | 导入Excel数据 | schema, key_column, update_columns, data_rows | bool, msg, count |
| execute_multi_column_update() | 执行更新 | schema, target_table, key_column, update_columns | success_count, fail_count, failed_records |
| rollback() | 回滚操作 | schema | bool, msg |
| cleanup_temp_table() | 清理临时表 | schema | bool |
| set_progress_callback() | 设置进度回调 | callback | 无 |
| get_backup_info() | 获取备份信息 | 无 | dict |

#### 2.3.3 更新流程

```
1. 验证表和列存在性
   ↓
2. 创建备份表 (原表名_BAK_时间戳)
   ↓
3. 创建临时表
   ↓
4. 导入Excel数据到临时表
   ↓
5. 逐行匹配更新主表
   ↓
6. 记录成功和失败数量
   ↓
7. 清理临时表
   ↓
8. 返回更新结果
```

## 4. 异常处理

### 4.1 异常分类

| 异常类型 | 说明 | 处理方式 |
|---------|------|---------|
| 数据库连接异常 | 连接失败、超时 | 提示用户检查配置 |
| Excel解析异常 | 文件格式错误、数据格式错误 | 提示用户检查文件 |
| 数据验证异常 | 重复键、类型不匹配 | 记录日志，停止操作 |
| 更新执行异常 | 权限不足、约束冲突 | 记录失败记录，回滚数据 |
| 文件IO异常 | 读写失败 | 提示用户检查文件权限 |

### 4.2 异常处理策略

1. 捕获所有可能的异常
2. 记录详细日志
3. 给用户友好的提示
4. 必要时执行回滚
5. 清理临时资源

## 5. 性能优化

### 5.1 优化策略

| 优化点 | 措施 |
|-------|------|
| Excel解析 | 使用openpyxl的read_only模式，流式处理 |
| 数据导入 | 批量插入，每批100条记录 |
| 数据更新 | 批量提交，减少数据库交互 |
| 界面响应 | 使用多线程处理耗时操作 |
| 日志写入 | 异步写入，不阻塞主线程 |

### 5.2 多线程设计

- 主线程：负责GUI渲染和用户交互
- 工作线程：负责数据库连接、Excel解析、数据更新等耗时操作
- 通过回调机制更新进度和界面
